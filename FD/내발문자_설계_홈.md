# 내발문자 설계 - 홈

## 역할
- 내발문자의 첫 진입 화면
- `지금 핫한 곳`과 `내 주변 실시간 상태`를 가장 빠르게 보여주는 첫 화면
- 사용자가 지금 갈지, 기다릴지, 피할지 결정하게 만드는 확인 화면
- 지도와 3초 공유로 사용자를 보내는 분기점

## 핵심 목표
- 앱을 켜자마자 “오, 지금 여기 상황 바로 보이네?”라는 인상을 만든다.
- 사용자가 “지금 어디가 살아있는지” 바로 이해하게 만든다.
- `지금 이 지역 상황`과 주변 관광/생활 장소 혼잡도를 먼저 보여준다.
- `오늘 행사`는 일정 정보가 아니라 현재 상태 진입점으로 보여준다.
- 사용자가 바로 `지도 보기` 또는 `기록하기`로 이동할 수 있어야 한다.

## 주요 구성
- 상단 요약 헤더
  - `내 발이 남긴 문장, 다음 발걸음의 선택이 된다` 가치 제안
  - 지도 진입 CTA
  - 기록 진입 CTA
- 실시간 상태 요약
  - 지금 핫한 곳
  - 오늘 행사 중 현재 붐비는 곳
  - 내 주변 장소 상태
  - 붐빔/보통/한산 색상 요약
  - 최근 상황공유 카드
  - 상태 분포 또는 대표 상태
- 지도 허브 안내
  - 지도에서 전체 상황 보기
- 행사 상태 요약
  - 오늘 행사
  - 지금 가장 붐비는 행사
  - 대기/입장/주변 혼잡 같은 현재 상태
- 보조 커뮤니티 흐름
  - 필요 시 소식 미리보기

## 노출 비율 원칙
- 초기 화면 비중은 `지도/상태/핫플레이스 75~90%`, `공식 정보/커뮤니티 10~25%`를 기준으로 둔다.
- 소식 커뮤니티 피드는 홈의 하단 보조 섹션으로 둔다.
- 홈은 홍보성 랜딩이 아니라 현재 지역의 현장 상태를 읽고 선택을 바꾸는 시작점처럼 보여야 한다.

## 제외 원칙
- 홈을 일반적인 홍보 랜딩처럼 만들지 않는다.
- 긴 설명, 과도한 프로모션 배너, 무관한 카드 나열은 줄인다.
- 소식 피드는 홈의 주인공이 아니라 보조다.
- 찜 목록 중심 첫 화면으로 만들지 않는다.
- 커뮤니티 글 중심 첫 화면으로 만들지 않는다.

## 데이터 축
- 사용자 데이터
  - 상황공유
  - 최근 기록
  - 최근 30분 기준 장소 상태
  - 태그 빈도
- 공공데이터
  - 공식 행사 위치와 일정
  - 행사 현장 상태를 만들기 위한 기본 장소/이벤트 메타
  - 관광공사 기반 관광 정보 요약

## 구현 관점
- `app/page.tsx`는 화면 조립을 담당한다.
- 실시간 상태, 공식 소식, 커뮤니티 미리보기는 각 feature 단위 컴포넌트로 유지한다.
- 홈에서 보여주는 공식 데이터는 TourAPI 기반 공식 데이터 우선, 실패 시 fallback 데이터를 사용한다.
- 단, 홈의 주인공은 행사 정보가 아니라 `행사/장소의 현재 상태`다.
- 첫 화면에는 `오늘 행사`, `지금 가장 붐비는 곳`, `근처 한산한 곳`을 상태 카드로 배치한다.

## 성공 기준
- 첫 화면만 보고도 `지금 상태를 확인하는 서비스`라는 인상이 전달된다.
- 첫 화면만 보고도 `지금 가장 핫한 곳을 보여주는 서비스`라는 인상이 전달된다.
- 지도 진입 의도가 바로 생긴다.
- 사용자가 “이거 보고 결정해야겠다”고 느낀다.
- 행사 정보가 일정 앱처럼 보이지 않고, 현재 상태 확인으로 이어진다.
- 홈에서 본 소식/행사/상태가 `지도에서 확인`, `기록 남기기`, `나의 내발문자에 기억`으로 이어지는 다음 행동을 가진다.

## 핵심 흐름 연결
- 홈은 `발견` 화면이다.
- 사용자는 홈에서 오늘의 단서를 발견한 뒤 다음 중 하나로 자연스럽게 이동해야 한다.
  - `소식 상세`: 맥락을 더 읽는다.
  - `지도`: 위치와 현재 상태를 확인한다.
  - `기록`: 내가 본 것 또는 현장 상태를 남긴다.
  - `내발문자`: 이미 기억한 장소를 다시 본다.
- 홈의 각 카드에는 가능한 한 하나 이상의 후속 행동이 있어야 한다.
  - 행사/장소 카드: `지도에서 보기`
  - 상태 카드: `상황이 달라요` 또는 `현장 공유`
  - 소식 카드: `자세히 보기`
  - 기억 카드: `나의 내발문자에서 보기`

## 관광자료 활용 방향
- 내발문자는 전국 유명 관광지 랭킹을 보여주는 서비스보다, 사용자의 현재 위치와 가까운 지역 관광 자원을 더 잘 활용하게 만드는 서비스에 가깝다.
- 따라서 홈에서 보여주는 공식 관광 자료도 `멀리 있는 대표 명소`보다 `지금 내 주변에서 바로 참고할 수 있는 행사, 장소, 생활권 정보`를 우선한다.
- 이 원칙은 내발문자의 지역 밀착형 정체성을 강화하고, 관광과 생활이 겹치는 실제 사용 맥락을 더 잘 반영한다.
- 행사 자료는 일정 소개가 아니라 현장 상태 공유를 발생시키는 진입점으로 사용한다.
