# 동플 설계 - 주변 순간들

## 역할
- `주변 순간들`은 지도 화면 하단 바텀시트에 노출되는 실시간 상태 카드 목록이다.
- 사용자는 이 영역에서 내 주변 장소와 행사의 현재 상태를 빠르게 훑고, 맞는 정보는 확인하고, 다른 상황이면 새로 제보한다.
- 이 영역은 커뮤니티 피드가 아니라 `지도 기반 실시간 판단 패널`이다.
- 핵심 문장은 `지금 여기 어떤 상태야?`를 3초 안에 이해하게 만드는 것이다.

## 제품 포지션
- 동플의 핵심 포지션은 `타이밍을 알려주는 앱`, `확인하는 앱`, `지금 어디가 살아있는지 보여주는 앱`이다.
- `주변 순간들`은 이 포지션을 지도 위에서 가장 자주 만나는 소비 화면이다.
- 사용자는 글을 읽으러 오는 것이 아니라 지금 갈지, 기다릴지, 피할지 결정하러 온다.

## 표시 대상
- 일반 장소 상태: 카페, 식당, 공원, 기타 현장
- 행사 현장 상태: 공식 행사와 `event_id`로 연결된 사용자 제보
- 확인 요청 상태: 누군가 지금 상태 확인을 요청한 장소

## 카드 정보 구조
- 카테고리
- 업데이트 시간
- 장소명 또는 행사명
- 현재 상태: 붐빔, 보통, 한산, 확인 요청
- 확인 수
- 짧은 현장 메모
- 상태 변경/검증 액션

## 카드 문구 원칙
- 정보 앱처럼 보이는 문구보다 행동을 바로 이해할 수 있는 문구를 쓴다.
- `정보 업데이트`처럼 추상적인 표현은 피한다.
- `제보`라는 단어는 위치와 대상이 명확할 때만 쓴다.
- 같은 카드 안에서 `업데이트`와 `제보`가 동시에 보이면 사용자가 차이를 이해하기 어렵다.

## 권장 명칭
- 영역 제목: `주변 순간들`
- 보조 라벨: `내 주변 실시간 상태`
- 빈 상태 문구: `아직 주변 순간이 없습니다. 첫 현장 공유를 남겨보세요.`
- 지도 링크 문구: `지도에서 주변 상태 보기`

## 액션 정의

### 1. 맞아요
- 의미: 현재 카드의 상태가 지금도 맞다고 확인한다.
- 대상: 이미 올라온 상태 카드
- 동작:
  - `verifyStatusWithTrust(statusId, userId)` 호출
  - `verified_count` 증가
  - 같은 사용자는 중복 확인되지 않도록 처리
- 버튼 문구:
  - `맞아요`
  - 보조 표기: `1명이 확인`, `3명이 확인`

### 2. 상황이 달라요
- 의미: 기존 카드의 상태가 지금 현장과 다르므로 새 상태를 남긴다.
- 대상: 이미 올라온 상태 카드
- 동작:
  - `liveCreate`를 `mode: "share"`로 연다.
  - 장소명, 주소, 좌표를 기존 카드에서 미리 채운다.
  - 사용자는 붐빔/보통/한산과 짧은 메모만 고른다.
- 버튼 문구:
  - `상황이 달라요`
  - 대안: `다른 상황 제보`
- 현재 `정보 업데이트` 버튼은 이 의미로 해석하는 것이 가장 자연스럽다.
- `정보 업데이트`라는 이름은 FD 방향과 맞지 않으므로 `상황이 달라요`로 바꾸는 것을 권장한다.

### 3. 현장 공유
- 의미: 선택한 위치나 현재 지도 중심 위치에 새 상태를 남긴다.
- 대상:
  - 지도에서 찍은 위치
  - 아직 카드가 없는 장소
  - 사용자가 새로 공유하고 싶은 장소
- 동작:
  - `liveCreate`를 `mode: "share"`로 연다.
  - 위치가 있으면 주소/좌표를 미리 채운다.
  - 행사 핀에서 진입하면 `event_id`를 함께 넘긴다.
- 버튼 문구:
  - `현장 공유`
  - 위치가 명확할 때: `여기에 현장 공유`
- 지도에서 새 마커를 찍은 직후에는 `여기는요!`를 사용한다.
- `여기는요!`는 "여기 지금 상태는 이래요"를 직접 알려주는 현장 공유 액션이다.
- 현재 `여기에 제보` 버튼은 이 의미로 해석하는 것이 맞다.
- 카드 내부에서는 `여기에 제보`보다 `현장 공유`가 더 명확하다.

### 4. 확인 요청
- 의미: 사용자가 직접 상태를 모르지만 누군가 확인해주길 요청한다.
- 대상:
  - 정보가 비어 있는 장소
  - 오래된 상태만 있는 장소
  - 방문 전 확인하고 싶은 행사/장소
- 동작:
  - `liveCreate`를 `mode: "request"`로 연다.
  - `is_request: true` 상태로 저장한다.
- 버튼 문구:
  - `확인 요청`
  - 대안: `이곳 확인 요청`
- 지도에서 새 마커를 찍은 직후에는 `여기는요?`를 사용한다.
- `여기는요?`는 "여기 지금 어떤가요?"를 묻는 확인 요청 액션이다.
- 기존 카드의 상태를 수정하는 의미로 쓰면 혼란이 생기므로 `정보 업데이트` 대신 쓰지 않는다.

## 새 마커 액션 문구
지도에서 사용자가 새 위치를 찍으면 아직 해당 위치에 카드 맥락이 충분하지 않다.
이때는 `확인 요청`, `현장 공유`처럼 기능 설명형 문구보다 가볍고 즉각적인 대화형 문구가 더 적합하다.

### 여기는요?
- 의미: 이 위치의 지금 상태를 다른 사용자에게 묻는다.
- 내부 동작: `liveCreate`를 `mode: "request"`로 연다.
- 저장 의미: `is_request: true`
- 추천 아이콘: `HelpCircle`
- 사용 위치: 지도에서 새 마커를 찍은 직후
- 사용하지 않을 위치: 이미 상태 카드가 있는 펼침 카드 내부

### 여기는요!
- 의미: 내가 이 위치의 지금 상태를 직접 알려준다.
- 내부 동작: `liveCreate`를 `mode: "share"`로 연다.
- 저장 의미: `is_request: false`
- 추천 아이콘: `Plus` 또는 `RadioTower`
- 사용 위치: 지도에서 새 마커를 찍은 직후
- 사용하지 않을 위치: 기존 상태가 틀렸다고 정정하는 카드 내부

## 새 마커와 기존 카드의 구분
- 새 마커는 아직 맥락이 없으므로 `여기는요? / 여기는요!`를 사용한다.
- 기존 상태 카드는 이미 정보가 있으므로 `맞아요 / 상황이 달라요 / 상세 보기`를 사용한다.
- 행사 핀은 행사 맥락이 있으므로 `행사 현장 공유 / 지금 상태 보기`를 사용한다.
- 같은 `share` 동작이라도 진입 맥락에 따라 문구를 다르게 쓴다.

## 현재 화면 해석
스크린샷 기준 카드 하단의 두 버튼은 현재 이렇게 이해된다.

- `정보 업데이트`
  - 실제로는 `mode: "request"`로 열릴 가능성이 높다.
  - 사용자 입장에서는 `정보를 수정하는 버튼인지`, `확인을 요청하는 버튼인지`, `새 제보를 남기는 버튼인지` 알기 어렵다.
  - FD 기준으로는 `확인 요청` 또는 `상황이 달라요` 중 하나로 명확히 갈라야 한다.

- `여기에 제보`
  - 실제로는 `mode: "share"`로 열리는 새 현장 공유 버튼이다.
  - 의미는 맞지만 표현이 투박하다.
  - `현장 공유` 또는 `여기에 현장 공유`가 더 자연스럽다.

## 권장 UI 구성
기존 상태 카드가 펼쳐졌을 때:

1. `맞아요`
2. `상황이 달라요`
3. `상세 보기`

지도에서 새 위치를 찍었을 때:

1. `여기는요?`
2. `여기는요!`

행사 핀에서 진입했을 때:

1. `행사 현장 공유`
2. `지금 상태 보기`

## 데이터 연결
- 기본 테이블: `live_status`
- 핵심 필드:
  - `place_name`
  - `category`
  - `status`
  - `status_color`
  - `is_request`
  - `verified_count`
  - `latitude`
  - `longitude`
  - `message`
  - `event_id`
  - `created_at`
  - `expires_at`
- 검증 테이블: `status_verifications`
- 행사 연결:
  - 공식 행사 상태는 `event_id` 기준으로 묶는다.
  - 행사 정보 자체보다 `지금 붐빔`, `대기`, `분위기`, `몇 분 전`을 우선 노출한다.

## 정렬 및 만료
- 최근 30분 데이터를 우선한다.
- 만료된 상태는 기본 목록에서 제외한다.
- 같은 장소의 여러 제보는 최신 상태와 최근 히스토리로 묶는다.
- 검증 수가 많고 최근성이 높은 카드를 위로 올린다.
- 요청 상태는 너무 오래 남아 있으면 정보 품질을 떨어뜨리므로 별도 만료 시간을 둔다.

## 성공 기준
- 사용자가 카드 버튼을 보고 1초 안에 다음 행동을 이해한다.
- `정보 업데이트`처럼 기능 의미가 불분명한 문구가 남아 있지 않다.
- 새 현장 공유가 3초 안에 가능하다.
- 상태 검증과 새 제보가 서로 다른 행동으로 분명히 구분된다.
- 행사 카드는 일정 앱처럼 보이지 않고 현장 상태 확인으로 이어진다.

## 구현 메모
- 현재 지도 바텀시트 컴포넌트는 `src/features/map/components/MapBottomSheet.tsx`이다.
- 현재 새 공유 진입은 `onOpenCreate("share")`로 연결된다.
- 현재 확인 요청 진입은 `onOpenCreate("request")`로 연결된다.
- 다음 구현에서는 버튼명을 먼저 정리한다.
  - `정보 업데이트` -> `상황이 달라요` 또는 `확인 요청`
  - `여기에 제보` -> `현장 공유`
- 새 마커에서는 `onOpenCreate("request")` 버튼을 `여기는요?`로 표시한다.
- 새 마커에서는 `onOpenCreate("share")` 버튼을 `여기는요!`로 표시한다.
- 기존 카드 안의 `request` 액션은 제품 의미상 어색하므로, 카드 내부에서는 `상황이 달라요`를 기본으로 두고 빈 위치/새 위치에서만 `확인 요청`을 노출하는 편이 좋다.
