# 내발문자 설계 - 지역 확장

## 목적
- 현재 수원 중심 구조를 다른 지역으로 확장할 수 있도록 공통 설계 포인트를 정의한다.
- 지역별 론칭이 늘어나도 제품 구조, 데이터 구조, 브랜드 메시지가 흔들리지 않게 한다.

## 기본 원칙
- `내발문자`는 공통 코어 제품이다.
- 지역은 `운영 단위`이자 `큐레이션 단위`로 다룬다.
- 특정 지역의 표현은 기본값일 뿐, 전역 규칙이 되면 안 된다.
- 사용자는 기본적으로 `현재 접속 지역 중심 서비스`를 경험한다.

## 브랜드 전략

### 1. 코어 브랜드
- 기본 브랜드는 `내발문자`
- 모든 공통 기능은 같은 제품 구조를 공유한다.
  - 홈
  - 지도
  - 기록
  - 소식
  - 나의 내발문자

### 2. 지역 테마 브랜드
- 론칭/캠페인/프로모션 단위로 지역 테마를 붙인다.
- 예시
  - `내발문자: 수원에 가다`
  - `내발문자: 강원삼척에 가다`
  - `내발문자: 전주한옥마을에 가다`

### 3. 브랜드 적용 방식
- 앱 이름 자체를 지역별로 쪼개기보다, 같은 제품 안에서 지역 테마를 얹는 방식이 안정적이다.
- 지역 테마는 아래에 반영한다.
  - 홈 상단 카피
  - 공식 소식 섹션
  - 추천 장소/기록 가이드
  - 이벤트/축제 큐레이션
- 지역 테마는 상시 전환 UI보다 `배너`, `특집 페이지`, `캠페인 진입점`에서 쓰는 것이 자연스럽다.

## 제품 구조 확장 포인트

### 1. 지역 컨텍스트를 전역 상태로 관리
- 현재 위치와 별개로 `활성 지역(active region)` 개념이 필요하다.
- 활성 지역은 다음 정보를 가진다.
  - 지역 ID
  - 지역명
  - 대표 중심 좌표
  - TourAPI areaCode / sigunguCode
  - 기본 지도 줌
  - 지역 대표 카피
  - 지역 테마 색상 또는 비주얼 키

### 1-1. 기본 동작 원칙
- 활성 지역의 기본값은 `현재 접속 지역`이다.
- 수원 사용자가 접속하면 수원 전용 서비스처럼 느껴져야 한다.
- 삼척 사용자가 접속하면 삼척 전용 서비스처럼 느껴져야 한다.
- 즉 다른 지역의 주요 행사나 글이 메인에 섞여 보이면 안 된다.

### 2. 현재 위치와 활성 지역을 분리
- 현재 위치
  - 사용자의 실제 GPS 위치
- 활성 지역
  - 지금 탐색/운영 중인 지역
- 예시
  - 사용자는 서울에 있지만 `내발문자: 강원삼척에 가다` 테마를 탐색할 수 있다.
- 다만 이 경우도 기본 UX는 위치 중심이며, 다른 지역 탐색은 예외적 상황으로 다룬다.

### 3. 지역별 홈 큐레이션
- 홈은 공통 구조를 유지하되, 지역별로 아래만 바뀐다.
  - 대표 문구
  - 공식 행사/축제
  - 추천 탐색 카테고리
  - 지역 특화 CTA
- 중요한 점은 `다른 지역 정보가 메인에 노출되지 않는 것`이다.
- 수원 접속자는 수원 이야기만, 삼척 접속자는 삼척 이야기만 우선 보게 만든다.

### 4. 지도는 지역 중심값을 교체 가능해야 함
- 지도 초기 중심값을 하드코딩하지 않는다.
- 지역 설정에 따라 아래가 교체되어야 한다.
  - 초기 중심 좌표
  - 기본 검색 범위
  - 공식 행사 기본 조회 범위
  - 추천 관광지 반경
- 현재 위치 기반으로 자동 세팅되는 것이 기본이며, 수동 전환은 보조 기능이어야 한다.

## 데이터 설계 포인트

### 1. 모든 주요 데이터에 지역 키를 붙일 준비
- 상황공유
  - region_id
  - area_code
  - sigungu_code
- 소식
  - region_id
  - category + region scope
- 공식 이벤트
  - area_code
  - sigungu_code
  - source region
- 나의 내발문자
  - 기록 생성 지역
  - 재방문 지역

### 1-1. 글과 커뮤니티의 지역성
- 소식 글도 같은 원칙을 따른다.
- 별도 지역 커뮤니티를 완전히 쪼개기보다, 하나의 소식 시스템 안에서 `지역 필터`를 강하게 적용한다.
- 사용자는 자기 지역 글만 우선 보게 되고, 필요할 때만 다른 지역 글을 본다.

### 2. 지역 메타 테이블 고려
- 향후 `regions` 테이블 또는 설정 파일이 필요하다.
- 최소 항목
  - id
  - slug
  - display_name
  - short_name
  - area_code
  - sigungu_code
  - center_lat
  - center_lng
  - hero_copy
  - theme_label
  - is_launch_region

### 3. 공공데이터 조회 단위 표준화
- TourAPI는 지역 확장 시 `areaCode`, `sigunguCode`가 핵심이다.
- 따라서 서비스 내부도 지역 조회 함수가 같은 파라미터 구조를 가져야 한다.
- 예시
  - `fetchOfficialEvents({ areaCode, sigunguCode })`
  - `fetchNearbyPlaces({ areaCode, sigunguCode, center, radius })`

## UX 설계 포인트

### 1. 지역을 고르는 입구
- 사용자의 기본 진입은 `현재 위치 기반`이다.
- 별도의 지역 전환을 메인 행동으로 두지 않는다.
- 다른 지역 진입은 아래 정도로 제한한다.
  - 특집 배너
  - 캠페인 페이지
  - 여행 기획전
  - 검색 결과 기반 진입

### 2. 지역 테마가 바뀌어도 UX 구조는 유지
- 수원이든 삼척이든 메뉴 구조는 바뀌지 않는다.
- 달라지는 것은 `무엇을 강조하느냐`다.

### 3. 지역별 성격 차이를 큐레이션에 반영
- 수원
  - 역사/도심/행사/먹거리 밀집
- 강원삼척
  - 해변/자연/계절성/이동 동선
- 따라서 같은 내발문자라도 지역별 핵심 카테고리와 추천 문구는 달라져야 한다.

### 4. 지역 전환 피로 최소화
- 너무 잦은 지역 전환을 강요하지 않는다.
- 현재 지역이 기본이고, 다른 지역은 필요할 때만 탐색하게 한다.
- 최근 본 지역, 여행 중 지역, 현재 지역을 구분해 보조적으로 제안할 수 있다.

## 운영 설계 포인트

### 1. 지역 론칭 단위
- 한 지역을 열 때 필요한 최소 세트
  - 지역 메타 설정
  - TourAPI 코드 매핑
  - 대표 홈 카피
  - 공식 행사 기본 노출 규칙
  - 추천 카테고리

### 2. 지역별 초기 데이터 전략
- 지역 론칭 초반에는 사용자 제보가 적을 수 있다.
- 따라서 아래 비율을 조절한다.
  - 공식 데이터 비중 상향
  - 추천 장소/공식 행사 선노출
  - 사용자 제보가 쌓이면 상황공유 비중 확대
- 소식 글도 지역 필터 기준으로 노출하되, 초기에는 공식/추천 콘텐츠가 빈 화면을 메워줄 수 있어야 한다.

### 3. 지역 특화 캠페인
- `내발문자: 수원에 가다`는 지역 특화 캠페인 이름으로도 활용 가능하다.
- 공모전, 제휴, 관광 시즌 프로모션에서 강한 메시지로 쓸 수 있다.
- 이런 지역 테마는 상시 네비게이션보다 `배너`나 `특집 페이지`로 표현하는 편이 더 자연스럽다.

## 현재 코드 기준 주의 포인트
- 지금은 일부 수원 고정값이 남아 있다.
- 향후 지역 확장 전 우선 정리할 대상
  - `locationStore` 기본 좌표/주소
  - 뉴스 상단의 `수원시 정자동 기준` 문구
  - 요약/브리핑 문구 속 수원 예시
  - mock 데이터의 수원 중심 샘플
  - 입력 placeholder의 특정 지역 예시

## 권장 구현 순서
1. `active region` 개념 도입
2. 지역 메타 설정 구조 도입
3. TourAPI 조회 함수 파라미터화
4. 홈/지도/소식의 지역 하드코딩 제거
5. 글/소식 지역 필터 기본 적용
6. 지역 테마 카피 시스템 도입
7. `내발문자: 수원에 가다` 형식의 지역 캠페인 적용

## 성공 기준
- 수원 외 지역을 붙여도 제품 구조를 다시 짜지 않아도 된다.
- 지역별 테마를 얹어도 브랜드는 여전히 `내발문자`로 유지된다.
- 지역마다 다른 관광 성격을 같은 제품 안에서 자연스럽게 표현할 수 있다.
- 사용자는 별도 전환 없이도 자기 지역 전용 서비스처럼 느낀다.

## 지역 관광자료 활용 원칙
- 내발문자의 공공 관광자료 활용 방향은 `전국 단위 유명 지역 소비`보다 `가까운 내 지역 활용`에 가깝다.
- 즉 사용자가 삼척에 있으면 삼척의 행사, 장소, 생활권 정보가 먼저 보여야 하고, 수원에 있으면 수원의 정보가 먼저 보여야 한다.
- 지역 확장도 전국 유명지 카탈로그를 늘리는 방식이 아니라, 각 지역 사용자가 자기 주변 관광 자원을 더 잘 발견하고 활용하게 만드는 방식으로 설계한다.
