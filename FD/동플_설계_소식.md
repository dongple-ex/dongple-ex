# 동플 설계 - 소식

## 역할
- 동네 커뮤니티 공간
- 실시간 제보가 아닌, 설명과 맥락이 있는 글을 나누는 게시판

## 핵심 목표
- 사용자는 질문, 추천, 경험, 생활 정보를 글로 남길 수 있다.
- 댓글과 공유를 통해 커뮤니티 상호작용이 이어진다.
- 공식 소식과 일반 소식의 차이를 명확히 구분한다.

## 주요 구성
- 상단 소개
  - 소식란의 성격 설명
- 카테고리별 피드
  - 주제별 캐러셀 또는 목록
- 게시글 상세
  - 본문
  - 댓글
  - 좋아요/신고
- 기록 버튼 연동
  - 기본 탭은 `소식 작성`

## 상호작용 구조
- 게시글 상세에는 댓글, 좋아요, 신고 흐름이 이어진다.
- 익명과 닉네임 작성이 모두 가능하되, 작성 방식은 기록 허브와 같은 규칙을 따른다.
- 소식은 상태보드처럼 즉시 갱신보다, 읽고 참여하는 흐름이 더 중요하다.

## 공식 소식과의 관계
- 공식 소식은 관광공사 또는 공공데이터 기반 정보
- 소식란은 사용자 커뮤니티 기반 글
- 같은 화면에 공존할 수 있지만 섹션과 라벨로 구분한다.

## 구현 원칙
- 소식은 `게시판`처럼 보여야 하며, `상황공유`처럼 보이면 안 된다.
- 공식 행사 데이터는 보조 정보나 별도 섹션으로 배치한다.
- 카테고리, 검색, 상세, 댓글 흐름이 하나의 피드 체계로 이어져야 한다.

## 콘텐츠 유형
- 동네질문
- 정보공유
- 같이해요
- 동네가게

## 데이터 축
- 사용자 데이터
  - 게시글
  - 댓글
  - 좋아요
- 공식 데이터
  - 행사/축제
  - 장소 정보 보강

## 성공 기준
- 소식란이 상황공유와 다른 성격이라는 점이 명확하다.
- 게시판다운 구조와 상호작용이 살아 있다.
- 공식 소식은 커뮤니티 글을 보완하는 정보로 작동한다.
