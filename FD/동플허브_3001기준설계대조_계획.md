# 동플허브_3001기준설계대조_계획

## 1. 목적
- 본 문서는 `localhost:3001`에서 확인되는 현재 동플 화면을 기준으로 실제 구현 상태를 설계 문서와 비교하고, 다음 수정 작업의 방향을 정리하기 위한 계획 문서다.
- 본 문서에서는 사용자가 별도로 띄운 `3000`, `3002` 포트는 검토 대상에서 제외한다.
- 판단 기준은 `동플허브_마스터스펙_구현.md`의 사용자 노출 원칙과 `지도 중심 허브`, `상황 우선 노출`, `공식 소식은 보조 축 유지` 원칙을 우선 적용한다.

## 2. 검토 기준 요약

### 마스터 스펙 기준 핵심 원칙
- 전면 경험은 `상황을 본다 → 제보한다 → 신뢰를 판단한다` 흐름으로 구성한다.
- 실시간 공유의 핵심 화면은 `지도(Map)`로 두고 위치 맥락에서 탐색과 제보를 수행한다.
- 홈은 사용자 참여형 상황 카드가 우선이며, 공식 이벤트/지역 소식은 보조 축으로 유지한다.
- 사용자가 홈에서 곧바로 지도 허브로 이동하고, 지도에서 제보·요청·검증 액션으로 이어져야 한다.

### 프론트 구조 기준 핵심 원칙
- `page.tsx`는 조립 역할을 맡고, 도메인별 실동작은 `features/*` 중심으로 정리한다.
- 홈, 지도, 상태 보드, 이벤트는 하나의 흐름으로 연결되어야 하며 서로 다른 제품처럼 보이면 안 된다.

## 3. 3001 기준 현재 구현 요약

### 확인된 현재 화면 구조
- 홈 상단에 `HeroSection`이 위치한다.
- 홈 핵심 콘텐츠로 `LiveBoardTickerv2`, `OfficialEventSection`, `QuestionSummaryV2`, 일반 `posts` 피드가 순서대로 배치되어 있다.
- 지도 페이지(`/map`)에는 지도, 카테고리 필터, 바텀시트, 실시간 상태 마커, 공식 이벤트 마커가 함께 구현되어 있다.
- 공식 이벤트 목록은 `fetchOfficialEvents()`를 통해 `events + events_ext` 구조를 반영하고 있다.

### 현재 구조의 장점
- 지도 허브가 실제로 존재하고, 실시간 상태와 공식 이벤트를 같은 공간 맥락에서 볼 수 있다.
- 실시간 상태 색상 규칙은 최근 정리된 공통 토큰을 통해 일관성이 높아졌다.
- 홈에 공식 소식 섹션이 분리되어 있어 보조 축 유지 방향은 일부 반영되었다.

## 4. 설계와의 차이

### A. 우선순위 상 가장 큰 차이
- 홈의 첫 인상과 주요 비중이 아직 `동네 상황 허브`보다 `V2 마케팅/프로모션 화면`에 가깝다.
- `HeroSection`의 2, 3번 슬라이드는 맛집/산책 코스 중심으로 흐르며, 마스터 스펙의 핵심 행동 흐름인 `상황 확인 → 제보 → 검증`과 직접 연결되지 않는다.
- 홈의 하단 메인 피드는 `posts` 중심으로 크게 편성되어 있는데, 현재 1차 구현 기준 문서에서는 `실시간 상황`과 `공식 이벤트`가 먼저 보여야 한다.

### B. 상태 보드 구현의 차이
- 실제 홈에서는 `LiveStatusBoard`가 아니라 `LiveBoardTickerv2`가 노출되고 있다.
- `LiveBoardTickerv2`의 `맞아요` 버튼은 아직 실제 검증 로직이 아니라 `alert("인증 완료!")` 임시 동작이다.
- 설계 문서 기준으로는 상태 보드가 데이터 신뢰 판단 흐름의 핵심인데, 현재 홈 노출 버전은 임시 홍보형 카드에 가깝다.

### C. 설계 범위를 벗어난 보조 요소 과다
- `QuestionSummaryV2`의 `AI 동네 브리핑`은 현재 고정 문구 기반이며, 마스터 스펙이나 1차 구현 범위의 핵심 축은 아니다.
- 사회적 증명 배너, 카테고리 그리드, 일반 피드가 현재 홈에서 차지하는 비중이 커서 실시간 상황 중심성이 약해진다.
- 결과적으로 홈이 `동플 허브 메인`보다 `확장형 커뮤니티 랜딩`처럼 보인다.

### D. 지도 허브 연계의 차이
- 지도 페이지 자체는 설계와 비교적 잘 맞지만, 홈에서 지도 허브로 사용자를 밀어주는 구조가 아직 약하다.
- 현재는 `/map` 링크가 존재하더라도, 홈 상단에서 바로 `우리 동네 상황을 지도에서 본다`는 메시지가 주축이 아니고 여러 성격의 CTA가 섞여 있다.

## 5. 수정 방향

### 5.1 홈의 역할 재정의
- 홈은 `동플 허브 메인 허브`로 재정의한다.
- 홈의 첫 화면은 아래 우선순위로 재배치한다.
  1. 상황 중심 Hero 또는 요약 헤더
  2. 실시간 상태 요약/티커
  3. 지도 허브 진입 CTA
  4. 공식 이벤트 요약
  5. 필요한 경우에만 보조 브리핑 또는 커뮤니티 피드

### 5.2 HeroSection 수정 방향
- 현재 3장 슬라이드형 마케팅 구성은 축소하거나 제거한다.
- Hero는 하나의 핵심 메시지로 단순화한다.
  - 예시: `지금 우리 동네 상황을 지도에서 바로 확인하세요`
  - 보조 문구: `이웃이 올린 실시간 제보와 공식 소식을 함께 봅니다`
- CTA는 `지도에서 보기`, `상황 공유하기` 두 축 정도로 고정한다.
- 맛집/산책/테마형 슬라이드는 현재 1차 범위에서는 제거 또는 하단 보조 영역으로 이동한다.

### 5.3 실시간 상태 영역 수정 방향
- 홈 메인 상태 노출 컴포넌트는 `LiveBoardTickerv2`와 `LiveStatusBoard` 중 하나로 정리한다.
- 권장 방향:
  - 홈 상단 요약형 컴포넌트는 유지하되, 실동작은 `LiveStatusBoard` 기준 로직으로 통합한다.
  - `맞아요`는 실제 `verifyStatusWithTrust` 기반 검증으로 연결한다.
  - `아니에요`는 다른 상태 제보 플로우로 연결한다.
- 즉, V2 전용 장식 UI는 남기더라도 상태 액션만큼은 MVP의 실제 도메인 로직을 사용해야 한다.

### 5.4 공식 이벤트 섹션 유지 방향
- `OfficialEventSection`은 유지한다.
- 다만 홈 내 비중은 실시간 상황보다 낮게 둔다.
- 공식 소식 섹션은 현재처럼 가로 스크롤 요약 카드 구조를 유지하되, 지도 허브와 연결되는 맥락 문구를 보강한다.

### 5.5 커뮤니티/브리핑 영역 조정 방향
- `QuestionSummaryV2`는 현재 하드코딩된 데모 성격이 강하므로 1차 메인에서 우선순위를 낮춘다.
- 일반 `posts` 피드는 메인 하단으로 이동하거나, 별도 `커뮤니티` 탭으로 위상을 내린다.
- 홈에서는 `지금 우리 동네는` 영역을 유지하더라도 `실시간 상황/공식 소식` 아래로 충분히 내려야 한다.

## 6. 파일별 수정 설계

### `src/app/page.tsx`
- 홈 조립 순서를 재구성한다.
- `HeroSection`, `LiveBoardTickerv2`, `OfficialEventSection`, `QuestionSummaryV2`, 일반 피드의 순서와 비중을 조정한다.
- `posts` 피드를 메인 핵심 축이 아닌 하위 보조 섹션으로 이동한다.

### `src/components/dashboard/v2/HeroSection.tsx`
- 다중 프로모션 슬라이드를 단일 핵심 메시지형 Hero로 단순화한다.
- CTA는 `/map` 중심으로 재정렬한다.
- 동플의 핵심 가치가 `동네 상황 지도 허브`라는 점이 문구와 시각 구조에서 바로 읽히도록 수정한다.

### `src/components/dashboard/v2/LiveBoardTickerv2.tsx`
- `맞아요` 버튼 임시 alert를 제거한다.
- 실제 상태 검증/업데이트 플로우와 연결한다.
- `LiveStatusBoard`와의 역할 중복을 정리하고, 가능하면 공통 상태 액션 훅 또는 서비스 로직을 참조하도록 정리한다.

### `src/features/status/components/LiveStatusBoard.tsx`
- 홈 노출형 요약 보드의 기준 컴포넌트로 재검토한다.
- 필요한 경우 `LiveBoardTickerv2`를 대체하거나, 최소한 액션 로직의 기준 소스로 삼는다.

### `src/components/dashboard/v2/QuestionSummaryV2.tsx`
- 메인 노출 우선순위를 하향 조정한다.
- 1차 구현 범위에서는 제거, 축소, 또는 추후 기능 플래그 처리 중 하나를 선택한다.

### `src/app/map/page.tsx`
- 현재 구조를 유지하되, 홈에서 넘어오는 사용자가 바로 `상황 탐색/공유` 흐름에 들어오도록 mode, focus, center 연계를 후속 검토한다.
- 지도는 현행 구현을 계속 메인 허브로 유지한다.

## 7. 권장 구현 순서
1. `page.tsx`에서 홈 IA를 먼저 재배치한다.
2. `HeroSection.tsx`를 설계 방향에 맞는 단일 메시지형으로 정리한다.
3. `LiveBoardTickerv2.tsx`의 임시 액션을 실제 상태 검증/제보 흐름으로 교체한다.
4. `QuestionSummaryV2.tsx`와 일반 피드의 우선순위를 낮추거나 분리한다.
5. 마지막으로 홈 → 지도 진입 메시지와 CTA 문구를 미세 조정한다.

## 8. 완료 기준
- 홈 진입 직후 사용자는 `우리 동네 실시간 상황을 본다`는 제품 정체성을 즉시 이해할 수 있다.
- 홈의 첫 번째 핵심 액션은 `지도에서 보기` 또는 `상황 공유하기`여야 한다.
- 실시간 상태 섹션은 임시 alert가 아닌 실제 데이터 액션을 수행해야 한다.
- 공식 이벤트 섹션은 보조 축으로 유지되되, 실시간 상황보다 위상이 낮아야 한다.
- `AI 브리핑`, 일반 피드, 프로모션 슬라이드가 실시간 상황 허브의 정체성을 흐리지 않아야 한다.

## 9. 메모
- 현재 `3001` 응답은 동플 프로젝트 화면으로 판단되지만, 메인 편성은 아직 `V2 전환 중간 상태`에 가깝다.
- 따라서 다음 작업은 단순 UI 보완이 아니라, 홈의 정보구조(IA)와 행동 흐름을 마스터 스펙 기준으로 다시 정렬하는 작업으로 보는 것이 맞다.
