# 내발문자 설계 - 지도

## 역할
- 내발문자의 핵심 화면
- 내 주변 실시간 상태를 확인하고 3초 안에 공유하는 중심 화면
- 행사 레이어와 일반 장소 레이어가 함께 작동하는 실시간 핫플레이스 지도
- 실시간 상황공유, 공식 행사 현재 상태, 위치 맥락 탐색이 한곳에서 이루어지는 허브

## 핵심 목표
- 사용자는 현재 위치 주변의 분위기와 혼잡도를 빠르게 파악한다.
- 사용자는 `지금 어디가 살아있는지`와 `지금 가장 핫한 곳`을 빠르게 파악한다.
- 사용자는 지도에서 바로 3초 상황공유를 남길 수 있다.
- 사용자는 공식 행사 정보가 아니라 행사 현장의 현재 상태를 같은 공간 맥락에서 본다.

## 주요 구성
- 지도 캔버스
  - 실시간 상황 마커
  - 장소별 붐빔/보통/한산 색상
  - 강조된 행사 마커
    - 혼잡도 색상
    - 최근 업데이트 시각
    - 현재 상태 뱃지
  - 일반 장소 마커
  - 클릭 위치 마커
- 지도 상단 인터랙션
  - 검색
  - 현재 위치
  - 카테고리 필터
    - 전체
    - 행사
    - 카페
    - 식당
    - 공원
- 하단 카드/시트
  - 선택한 장소의 상황 카드
  - 선택한 행사의 현재 상태 카드
  - 현재 상태와 최근 업데이트
  - 상위 태그
  - 클릭 위치에서 기록하기
  - 공식 행사 정보

## 세부 UX 원칙
- 지도 하단은 바텀시트 구조를 유지한다.
- 바텀시트는 드래그와 스냅 포인트를 지원한다.
  - 최소 보기
  - 중간 보기
  - 확장 보기
- 지도 마커 클릭 시 해당 카드로 자동 스크롤되도록 리스트와 연동한다.
- 카드 내부에서는 `맞아요`, `다른 상황 제보`, `상세 보기` 같은 즉시 액션이 가능해야 한다.
- 지도 클릭 후 생성된 위치는 바로 기록 흐름으로 이어질 수 있어야 한다.
- 공유 흐름은 혼잡도 선택 → 태그 선택 → 완료를 기본으로 하며, 최대 3초를 목표로 한다.
- 텍스트 입력은 선택 사항이고, 버튼 중심으로 구성한다.
- 행사 핀은 일정 정보보다 현재 상태 뱃지(`지금 붐빔`, `대기 김`, `입장 원활`)를 우선 보여준다.
- 행사 핀은 `몇 분 전` 업데이트인지 함께 보여준다.
- 행사 카드는 공공 API의 이름/위치/시간/설명을 기본 뼈대로 쓰고, 내발문자의 붐빔/대기/분위기 리포트를 주 정보로 보여준다.
- 일반 장소는 관광지 주변 카페/식당/공원처럼 평소에도 확인할 수 있는 방문 맥락 레이어로 유지한다.

## 사용자 흐름
1. 위치 확인
2. 지금 핫한 곳 탐색
3. 행사 또는 일반 장소 필터 선택
4. 특정 위치 선택
5. 상황공유 또는 공식 정보 확인
6. 선택 변경 또는 기록으로 연결

## 상태보드 연결 원칙
- 상태보드는 지도의 보조 요소가 아니라 지도 경험의 일부다.
- 목록 카드와 지도 마커는 같은 데이터를 바라본다.
- 검증, 업데이트, 상세 메모 확인이 같은 맥락에서 이어져야 한다.
- 상태는 최근 30분 데이터를 우선 반영한다.
- 태그는 빈도 기반으로 상위 항목을 노출한다.

## 공공데이터 연결
- 1차
  - `searchFestival1`
  - 공식 행사 마커, 행사 상태 카드의 기본 메타
  - `event_id` 기준 사용자 리포트 연결
- 2차
  - `locationBasedList1`
  - 주변 관광지 탐색
- 3차
  - `detailCommon1`, `detailImage1`
  - 장소 상세 확장

## 구현 원칙
- 지도는 단순 배경이 아니라 서비스 중심 인터페이스다.
- 상황공유 액션은 지도와 최대한 가깝게 둔다.
- 공식 행사는 사용자 제보를 발생시키는 강한 진입점으로 다룬다.
- 공식 행사 자체의 설명보다 `그 행사장의 현재 상태`를 우선한다.
- 공식 데이터는 Ground Truth 성격으로 사용하되, 사용자 현장 데이터가 더 즉시적인 층을 형성한다.
- 공공데이터는 뼈대이고, 내발문자 실시간 리포트는 판단을 만드는 뇌다.
- 초기 핵심 데이터는 `places`, `live_status`, `status_verifications`를 기준으로 모델링한다.
- 지도 표시에 필요한 장소별 현재 상태는 우선 최근 `live_status` 계산값으로 제공하고, 성능상 필요할 때 `place_status` 같은 캐시/요약 테이블을 추가한다.

## 성공 기준
- 사용자가 지도를 보자마자 “여기서 탐색하고 공유하는구나”를 이해한다.
- 사용자가 지도를 보자마자 “지금 어디가 핫한지 알 수 있구나”를 이해한다.
- 상황공유 작성이 3초 안에 가능하다.
- 상태가 공유 후 즉시 지도와 장소 카드에 반영된다.
- 공식 행사와 사용자 상황 데이터가 충돌하지 않고 공존한다.
- 소식에서 본 장소와 나의 내발문자에 저장한 장소가 지도에서 다시 확인 가능하다.

## 흐름 연결 원칙
- 지도는 `현재화` 화면이다.
- 소식 상세에서 들어온 장소는 지도에서 선택된 상태로 보여주는 것이 이상적이다.
- 내발문자에서 들어온 장소는 해당 위치로 이동하고, 최근 상태와 주변 순간을 함께 보여준다.
- 지도에서 기록한 상태는 즉시 나의 내발문자에 저장되어야 한다.
- 지도 카드에는 필요 시 `기억하기` 또는 `나의 내발문자에 저장` 액션을 둘 수 있다.
